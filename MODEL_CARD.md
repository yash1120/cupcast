# Model card — CupCast outcome & goals models

## Overview
| | |
|---|---|
| Task | 3-way match outcome (win/draw/loss) + expected goals |
| Outcome model | `HistGradientBoostingClassifier` wrapped in `CalibratedClassifierCV` (isotonic, 3-fold) |
| Goals model | `PoissonRegressor` on attacker-perspective rows (2 per match) |
| Training data | International A-matches 1990–present where both sides have ≥30 prior matches (~30k samples) |
| Features | 7: Elo diff (incl. home adv), home/away Elo, neutrality, competition K-factor, form diff, goal-diff form |
| Evaluation | Strict time split at 2024-01-01; metrics in `models/metrics.json` |

## Intended use
Entertainment / portfolio forecasting of international football. Probabilities are
calibrated at the population level but carry wide per-match uncertainty; not suitable
for betting decisions.

## Data lineage & leakage control
Features are generated by a single chronological replay of the match history: every
training row sees only ratings and form computed from *earlier* matches. The same
replay produces the prediction-time state (current Elo + form), so training and
serving share one code path (`features.py`).

## Known limitations & biases
- Results-only: no squads, injuries, weather, travel, or market information.
- Teams with sparse recent schedules (e.g. Curaçao's short history at this level)
  have noisier ratings.
- Draw probability is hardest to model (class ~24%); errors concentrate there.
- The 2026 expanded format has never been played — knockout dynamics (32-team
  bracket, possible 4th-substitute rules etc.) are extrapolated, not learned.

## Derived match detail
Scoreline grids (`/api/detail`) are independent Poissons region-rescaled to the
calibrated W/D/L probabilities. Over/unders, BTTS and clean-sheet figures are exact
sums over that grid — they inherit the classifier's calibration but assume goal
independence within each outcome region. Weather (Open-Meteo) is display context
only, never a model input.

## Retraining
`python scripts/download_data.py && python -m cupcast.train && python -m cupcast.simulate && python -m cupcast.weather`
— intended cadence during the tournament: daily.
